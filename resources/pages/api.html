# Web APIs and REST

Web APIs

An Application Programming Interface (API) is a software tool that lets developers interact with another application to use its functionality.

Types of APIs

  • Browser APIs: Access browser features like Geolocation API, audio, VR, etc. Permissions are often required.
  • Third-party APIs: Services like OpenWeather API offer data or functionality externally. Usually require an API key.

For more, see MDN’s Web API documentation.

Making Requests

To request data from an API, you often need to include:

  • Resource path (e.g. /weather)
  • HTTP method (e.g. GET, POST)
  • Headers (e.g. Authorization, Content-Type)
  • Parameters (e.g. city name, time, forecast type)

Details are always found in the API’s official documentation.

Response Format: JSON

Most APIs respond with data formatted in JSON:

{
  "temperature": {
    "celcius": 25
  },
  "city": "chicago"
}

REST Architecture

REST (REpresentational State Transfer) is an architectural style that enables stateless communication between clients and servers over HTTP.

Principles

  • Stateless: Each request contains all needed info; no session memory.
  • Resource-based: Resources are nouns (e.g. /customers), not actions.
  • Separation of Client and Server: Each can evolve independently.

HTTP Methods (CRUD)

  • GET Retrieve a specific resource or a collection
  • POST Create a new resource
  • PUT Update an existing resource
  • DELETE Remove a resource

More in this Codecademy article.

RESTful Paths

Paths identify resources clearly:

  • /customers → All customers
  • /customers/223/orders/12 → Order 12 for customer 223
  • POST /customers → Create new customer
  • GET /customers/:id → Retrieve customer by ID
  • DELETE /customers/:id → Remove customer by ID

Use plural nouns for resources. Avoid verbs in paths.

HTTP Status Codes

  • 200 OKStandard response for successful requests
  • 201 CREATEDResource was successfully created
  • 204 NO CONTENTSuccess, no content returned
  • 400 BAD REQUESTClient-side syntax or format error
  • 403 FORBIDDENClient lacks permission
  • 404 NOT FOUNDResource does not exist
  • 500 INTERNAL SERVER ERRORGeneric server failure

RESTful API Example – Customer Management

1. View all customers

GET http://fashionboutique.com/customers
Accept: application/json

2. Create a new customer

POST http://fashionboutique.com/customers
Content-Type: application/json

{
  "customer": {
    "name": "Scylla Buss",
    "email": "scylla.buss@codecademy.org"
  }
}

3. View a single customer

GET http://fashionboutique.com/customers/123
Accept: application/json

4. Update an existing customer

PUT http://fashionboutique.com/customers/123
Content-Type: application/json

{
  "customer": {
    "name": "Scylla Buss",
    "email": "scyllabuss1@codecademy.com"
  }
}

5. Delete a customer

DELETE http://fashionboutique.com/customers/123

  • Accept: application/jsonClient accepts JSON response
  • Content-Type: application/jsonBody is in JSON format
  • text/htmlHTML document
  • text/cssCSS stylesheet
  • image/pngPNG image
  • application/pdfPDF document
  • application/xmlXML data

# JSON

What is JSON?

JSON (JavaScript Object Notation) is a language-independent data format used to store and exchange data across systems. It is standardized by ECMA International.

JSON is widely used in web applications to send and receive data between clients (like browsers) and servers.

Public APIs such as Spotify or Google Maps often deliver data in JSON so that any programming language can parse it easily.

JSON Syntax

JSON is based on JavaScript object syntax. Here's an example:

{
  "student": {
    "name": "Rumaisa Mahoney",
    "age": 30,
    "fullTime": true,
    "languages": ["JavaScript", "HTML", "CSS"],
    "GPA": 3.9,
    "favoriteSubject": null
  }
}

JSON Data Types

  • stringText enclosed in double-quotes
  • numberInteger or float
  • objectName-value pairs enclosed in braces { }
  • arrayComma-separated values in square brackets [ ]
  • booleantrue or false
  • nullEmpty or unknown value

Dates are not supported directly in JSON. Use ISO 8601 string format: "2014-01-01T23:28:56.782Z"

JSON vs JavaScript Objects

Differences between JSON and native JavaScript objects:

JSON

{
  "person": {
    "name": "Kate",
    "age": 30,
    "hobbies": ["reading", "writing", "cooking", "tennis"]
  }
}

JavaScript

{
  person: {
    name: 'Kate',
    age: 30,
    hobbies: ['reading', 'writing', 'cooking', 'tennis']
  }
}
  • JSON requires double quotes for keys and string values.
  • JavaScript accepts both single and double quotes.

Reading JSON

In JavaScript, use JSON.parse() to convert a JSON string into an object:

const jsonData = '{ "book": { "name": "JSON Primer", "price": 29.99, "inStock": true, "rating": null } }';
const jsObject = JSON.parse(jsonData);
console.log(jsObject);

Output:

{
  book: { name: 'JSON Primer', price: 29.99, inStock: true, rating: null }
}

Writing JSON

Use JSON.stringify() to convert an object to a JSON string:

const jsObject = { book: 'JSON Primer', price: 29.99, inStock: true, rating: null };
const jsonData = JSON.stringify(jsObject);
console.log(jsonData);

Output:

{
  "book": "JSON Primer",
  "price": 29.99,
  "inStock": true,
  "rating": null
}

# fetch() with .then()

fetch()

The fetch() function is used to make HTTP requests to APIs in JavaScript. It:

  • Creates a request and sends it to a specified endpoint
  • Returns a Promise that resolves to a Response object
  • Allows chaining with .then() to handle success and error cases

Basic syntax:

fetch('https://api-to-call.com/endpoint')

.then() Chaining

The .then() method is chained to fetch() to handle the asynchronous response:

  • First .then(): checks if the response is OK and returns response.json()
  • Error callback: logs the network error if the endpoint is unreachable
  • Second .then(): receives the parsed jsonResponse and handles the data

fetch() - GET Example

fetch('http://api-to-call.com/endpoint')
  .then(response => {
    if (response.ok) {
      return response.json();
    }
    throw new Error('Request failed!');
  }, networkError => console.log(networkError.message)
).then(jsonResponse => {
  // Code to execute with jsonResponse
});

This fetch call retrieves data from the endpoint and handles both success and error cases.

fetch() - POST Example

POST requests require a second argument in fetch() with method and body:

fetch('http://api-to-call.com/endpoint', {
  method: 'POST',
  body: JSON.stringify({ id: '200' })
}).then(response => {
  if (response.ok) {
    return response.json();
  }
  throw new Error('Request failed!');
}, networkError => console.log(networkError.message)
).then(jsonResponse => {
  // Code to execute with jsonResponse
});

The body is stringified and sent as JSON. The response is handled exactly like in the GET request.

# fetch() with async / await

Why use async / await?

async / await provides a clear and readable way to perform asynchronous operations such as HTTP requests using fetch().

Compared to promise chaining (.then()), this syntax is easier to follow and debug.

response.json() is itself asynchronous and must be awaited.

GET Request

This example fetches data from a URL using async / await.

const getData = async () => {
  try {
    const response = await fetch('https://api-to-call.com/endpoint');
    if (response.ok) {
      const jsonResponse = await response.json();
      // Code to execute with jsonResponse
    }
    throw new Error('Request failed!');
  } catch (error) {
    console.log(error);
  }
};

POST Request

This example sends data to a server using POST and async / await.

const getData = async () => {
  try {
    const response = await fetch('https://api-to-call.com/endpoint', {
      method: 'POST',
      body: JSON.stringify({ id: 200 })
    });
    if (response.ok) {
      const jsonResponse = await response.json();
      // Code to execute with jsonResponse
    }
    throw new Error('Request failed!');
  } catch (error) {
    console.log(error);
  }
};

GET with Query String

You can dynamically construct query strings when using fetch() with async / await. Use URLSearchParams to safely encode parameters.

const getWeather = async () => {
  const params = new URLSearchParams({
    city: 'Naples',
    units: 'metric'
  });

  try {
    const response = await fetch(`https://api.example.com/weather?${params}`);
    if (response.ok) {
      const jsonResponse = await response.json();
      // Use jsonResponse data
    }
    throw new Error('Request failed!');
  } catch (error) {
    console.log(error);
  }
};

Output URL: https://api.example.com/weather?city=Naples&units=metric
