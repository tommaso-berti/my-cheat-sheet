JavaScript cheatsheet

This is a quick reference cheat sheet for understanding and writing JavaScript.

# Variables

Variable Declarations

  • var Used before ES6 to declare variables. It is function-scoped.
  • let Introduced in ES6; allows reassignment and is block-scoped. Can be declared without an initial value (defaults to undefined).
  • const Also introduced in ES6; defines a constant variable. Must be assigned at declaration and cannot be reassigned.
  • let vs var let is block-scoped whereas var is function-scoped, reducing the risk of scope pollution.

Interpolation

Template literals, introduced in ES6, allow you to embed variables and expressions directly within strings.

const myPet = 'armadillo';
console.log(`I own a pet ${myPet}.`);
// Output: I own a pet armadillo.
  • Simplify string construction and improve readability.
  • Enclose strings in backticks and embed variables or expressions using ${expression}.

Truthy and Falsy

In JavaScript, conditions check the truthiness of a variable rather than its explicit boolean value. A variable with a non-falsy value evaluates to true in a conditional context.

Falsy values

  • 0
  • Empty strings ("" or '')
  • null
  • undefined
  • NaN

Scope

  • Global Scope Variables declared outside any block are global. They reside in the global namespace and can be accessed from anywhere.
  • Block Scope Variables declared inside a block ({ ... }) are only accessible within that block. 
{
  let blockVar = 'I am block scoped';
  console.log(blockVar); // Works here
}
console.log(blockVar); // Error: blockVar is not defined

Other Concepts

  • typeof An operator that returns a string representing the type of a value.
  • Truthy / Falsy Values that evaluate to true or false in boolean contexts. Falsy values include 0, "", null, undefined, and NaN.
  • Ternary Operator A concise alternative to if...else. Syntax: condition ? expression_if_true : expression_if_false.
  • Switch A control statement that compares an expression to multiple cases. Use break to avoid fall-through.
